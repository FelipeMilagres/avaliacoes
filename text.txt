O desafio mobile foi desenvolvido utilizando WebdriverIO com Appium e apresenta uma estrutura organizada, aplicando o padrão Page Object e separação adequada entre pages, testes e configurações. O projeto demonstra bom entendimento do framework utilizado, incluindo execução local, integração com BrowserStack para dispositivos reais, geração de relatórios com Allure e pipeline de CI/CD utilizando GitHub Actions. Os cenários solicitados foram implementados e executaram com sucesso no ambiente Android, demonstrando estabilidade da automação. Como ponto de melhoria, a automação não executa corretamente no ambiente iOS devido ao uso de seletores específicos de Android, o que impede a execução multiplataforma esperada no desafio. Considerando os critérios avaliados, o desafio mobile foi aprovado com ressalvas.


Considerando os resultados dos desafios de API e Mobile, o candidato demonstrou bom entendimento das ferramentas utilizadas e capacidade de estruturar automações utilizando frameworks consolidados. No desafio de API apresentou uma automação funcional e organizada, com integração em pipeline e geração de relatórios, tendo como ponto de melhoria apenas a cobertura de validações mais ampla. No desafio mobile aplicou corretamente o padrão Page Object Model, com execução estável no Android e integração com CI/CD, embora a execução em iOS não tenha funcionado devido ao uso de seletores específicos de Android. Diante do conjunto das entregas, o candidato está aprovado para entrevista técnica.
