Resumo Executivo

Este documento apresenta a consolidação do levantamento realizado junto às squads com o objetivo de identificar os principais tipos de massas de teste utilizados nos cenários de validação funcional.

A análise foi conduzida como parte do épico Discovery de Automação de Massas, cujo objetivo é identificar oportunidades para automatizar a geração de dados utilizados nas atividades de teste.

Participaram do levantamento 21 squads, que descreveram os cenários que dependem de massas de teste, suas parametrizações e dependências sistêmicas.

A partir dessa análise foi possível:

identificar as massas mais recorrentes entre as squads;

compreender as parametrizações mais utilizadas;

mapear dependências sistêmicas relevantes para geração de dados;

identificar oportunidades de reutilização de massas entre cenários.

Essas informações servirão como base para definição do escopo inicial da automação de geração de massas de teste.

1. Objetivo do Documento

Este documento apresenta a consolidação do levantamento realizado junto às squads com o objetivo de identificar quais tipos de massas de teste são mais utilizados nos cenários de validação funcional.

O levantamento foi conduzido como parte do épico Discovery de Automação de Massas, cujo objetivo é identificar oportunidades para automatizar a geração de dados utilizados nas atividades de teste.

A partir desta análise foi possível:

identificar massas mais recorrentes entre as squads;

compreender as parametrizações mais utilizadas;

Essas informações servirão como base para definição do escopo inicial da automação de geração de massas de teste.

2. Contexto Atual

Durante a execução de testes funcionais, as squads dependem da disponibilidade de massas de teste configuradas com características específicas.

Atualmente, a geração dessas massas ocorre principalmente por meio de solicitações registradas na planilha UAT4. Após a geração das massas, os dados ficam disponíveis para consulta no Locker, que atua como repositório centralizado de massas utilizadas nos ambientes de teste.

Apesar da existência desse repositório centralizado, a criação de novas massas ainda depende de processos operacionais, o que pode gerar:

dependência de solicitações manuais;

tempo de espera para geração de massas.

Para apoiar o discovery de automação de massas, foi realizado um levantamento junto às squads solicitando a descrição dos cenários que dependem de massas de teste.

3. Escopo do Levantamento

O levantamento consolidou informações fornecidas por 21 squads. Cada time descreveu suas necessidades de massa utilizando uma planilha estruturada contendo as seguintes informações:

cenário;

parametrização da massa;

dependências;

resultado esperado;

indicação se permite validar outro cenário;

método de obtenção da massa.

Essas informações permitiram identificar padrões recorrentes na utilização de massas de teste dentro do ecossistema.

Essas parametrizações refletem diferentes estados de cliente que precisam ser simulados para execução dos testes.

6. Reutilização de Massas

A coluna "Permite validar outro cenário" indicou que diversas massas podem ser reutilizadas em diferentes cenários.

Em vários casos, as squads indicaram que determinadas massas permitem validar:

todos os cenários;

cenários relacionados ao mesmo fluxo;

cenários subsequentes dentro do fluxo do produto.

Isso demonstra que existe potencial para reutilização de massas entre diferentes cenários de teste.

7. Métodos Atuais de Obtenção de Massas

As squads indicaram diferentes formas de obtenção das massas utilizadas nos testes.

Entre os métodos identificados estão:

solicitação de massas via planilha UAT4;

utilização de funcionalidades do Corebank;

utilização de massas previamente existentes;

criação de massas via API.

O uso recorrente da planilha UAT4 indica que o processo atual ainda depende de interações operacionais para criação de novas massas.

10. Principais Gargalos Identificados no Processo Atual

A análise das informações fornecidas pelas squads também permitiu identificar alguns desafios no processo atual de geração e utilização de massas de teste.

10.1 Dependência de solicitações operacionais

Grande parte das massas utilizadas pelas squads precisa ser solicitada por meio da planilha UAT4, o que gera dependência de processos operacionais para criação de novos dados.

10.2 Tempo para obtenção de massas

Como a geração de massas depende de solicitações e processamento operacional, pode haver tempo de espera para disponibilização dos dados necessários para testes.

10.3 Dependência de múltiplos sistemas

A geração e validação das massas envolve integração com diversos sistemas do ecossistema, o que pode aumentar a complexidade do processo de criação de dados.

11. Uso do Levantamento para Automação de Massas

O principal objetivo deste levantamento é identificar quais massas devem ser priorizadas para automação.

Com base nas análises realizadas, é possível iniciar a priorização considerando:

massas mais utilizadas pelas squads;

massas que suportam múltiplos cenários;

massas com menor complexidade de geração.

A automação dessas massas pode gerar benefícios como:

redução do esforço operacional;

aumento da autonomia das squads;

maior velocidade na execução de testes.
